# 設計書（サンプル案件）: 赤ちゃんお祝いアルバムサイト

`要件定義_赤ちゃんお祝いアルバム.md`をコンテキストとして、ai-dev-council
（OpenAI、設計ステージ単体）に生成させた設計書。レビュー・実装はまだ
行っていない。

## 概要

『友達の赤ちゃん誕生祝い』の写真とメッセージを投稿できるアルバム
Webサイトを、Flask + SQLiteで設計する。全てローカル動作・シングル
ファイル展開で、誰でも投稿・閲覧可、設定値/文言はconfigファイルに
集約する。ギャラリー表示・投稿フォーム・画像保存・設定一元化が
ポイント。

## 要件

- 投稿フォーム：投稿者名・写真・お祝いメッセージを入力して送信できる
- ギャラリー：投稿された内容（写真・名前・メッセージ・日付）を新着順
  表示できる
- 投稿・閲覧時にログイン不要・認証無し
- アップロード写真はサーバ内フォルダへ保存、パスはDBに記録
- DBはSQLiteを利用（テーブル：投稿者名/メッセージ/ファイル名/投稿日時）
- 設定ファイル(config.py)一つに"赤ちゃん名/サイトタイトル/コピー/
  テーマカラー"を定義し、コードから参照する
- アップロード可能なファイル種別は画像のみ、必須項目未入力や不正
  ファイルの場合はエラー表示
- デザインはパステルのイエロー×ミント系を基調とし、CSSで装飾
- 外部API、外部サービス・クラウドDB等は使用不可

## アーキテクチャ

Python 3.x + FlaskのWebアプリケーション構成。DBはSQLite単一ファイル
（Flask組込サポート有）。設定値は専用config.pyからimportしFlaskアプリ
に適用。テンプレートはJinja2（標準）。静的ファイル（CSS、画像）は
static/配下、アップロード画像はuploads/配下保存。フォームデータは
WTForms等は使わずシンプルにPOSTパラメータで処理。ファイル拡張子で
画像判定。DBにはテーブル1つ（posts: id, name, message, filename,
timestamp）。ミス入力・不正ファイル時はフォーム下部にエラーメッセージ
出力。全機能はapp.pyに集中。デプロイ不要・ローカルrun前提。

## ファイル計画

| パス | 役割 |
|---|---|
| `app.py` | Flask本体/ルーティング/DBアクセス/投稿処理/ギャラリー表示/設定ファイル読み込み、全体ロジック管理 |
| `config.py` | 赤ちゃん名/サイトタイトル/キャッチコピー/テーマカラー（メイン・アクセント）の一元管理 |
| `templates/index.html` | トップページ：投稿フォーム＋ギャラリービュー表示のJinja2テンプレート |
| `static/css/style.css` | パステル系カラースキーム、全体レイアウトのCSS |
| `uploads/（ディレクトリ）` | ユーザーアップロード写真の保存先。app直下専用ディレクトリ |
| `tests/test_app.py` | 投稿・ギャラリー表示・バリデーション正常系/異常系テスト（pytest） |

## テスト方針

1. 投稿フォームに全項目入力し、正常にDBおよびuploads/に反映され
   ギャラリーで表示されるか確認
2. 投稿が新しい順に並ぶこと、画像・文言が一致するかテスト
3. 画像以外（例: txt, fake.jpg.exe等）の誤送信時、エラーを正しく
   表示し保存されないことをテスト
4. 必須項目（名前、メッセージ、写真）の未入力送信でエラー表示が出、
   DBに記録されないことを検証
5. 設定ファイル内文言（タイトル、色等）の改変後、再読み込みで即時
   反映されているか目視テストも実施
6. テストコード（pytest）にて自動化された正常系/異常系テストも記述
