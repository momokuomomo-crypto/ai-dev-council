# ホットドッグ提案CLI

ai-dev-council（設計→設計レビュー→Claude Agent SDKによる自律実装→
実装レビュー→issue記録）で自動生成した、対話形式のホットドッグレシピ
提案ツール。

プログラミングに詳しくない場合は、まず
[かんたんガイド](かんたんガイド.md)を読むこと。

## 使い方

```
python main.py
```

実行すると5つの質問が順番に表示されるので、番号を入力してEnterを押す。

1. 辛いものは好きですか？
2. 今日はこってり系とあっさり系、どちらの気分ですか？
3. 今日の気分に近いものはどれですか？
4. チーズは好きですか？
5. 好みの味の系統はどれに近いですか？

回答が終わると、その好みに合ったパン・ソーセージ・ソース・トッピングの
組み合わせと、材料リストが表示される。

## ファイル構成

- `main.py` — CLIエントリポイント（対話フロー・結果表示）
- `questions.py` — 質問定義と入力受付（異常入力時は再質問する）
- `hotdog_recipes.py` — パン・ソーセージ・ソース・トッピングのデータと選定ロジック
- `test_main.py` — pytestによるテスト（33件）

## テスト

```
pip install pytest
python -m pytest test_main.py -v
```

## 既知の軽微な指摘（実装レビューより、未対応）

- `hotdog_recipes.py`のデータが空の場合、`pick_one`がNoneを返しうるが
  未チェック（現状のデータでは発生しない）
- 「辛いものは苦手」を選んでも`mild`タグを持つ要素が無いため、
  「spicyを避ける」以上の効果を持たない
