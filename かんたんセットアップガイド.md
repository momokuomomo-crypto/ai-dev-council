# かんたんセットアップガイド

パソコンの「コマンド」を使ったことがない人でも、このツールを動かせる
ように、順番に説明します。むずかしい言葉が出てきたら、その場で説明します。

---

## このツールで何ができるの？

作りたいものを1文で説明すると、AIが「設計 → 設計の確認 → 実際にプログラム
を書く → 書いたものの確認 → 記録を残す」までを自動でやってくれます。

このツール（ai-dev-council）は「実際にプログラムを作る係」。話し合って
考えをまとめる係は、姉妹ツールの
[ai-council](https://github.com/momokuomomo-crypto/ai-council)
（そちらにも、かんたんセットアップガイドがある）。

- **ai-council** → 話し合って、考えをまとめる係
- **ai-dev-council**（このツール） → まとまった考えをもとに、実際に
  プログラムを作る係

この2つは別々のツールなので、それぞれ別に準備する必要がある。まず
このガイドだけでも、ai-dev-council単体で動かせる。

---

## 準備するもの（さきに全部見てから始めよう）

- Windowsのパソコン
- インターネット
- Python（パイソン）というソフト → ステップ2で無料でダウンロードする
- Node.js（ノードジェイエス）というソフト → ステップ4でダウンロードする
  （AIが実際にプログラムを書く機能を動かすために必要）
- Claude Code CLIというソフトへのログイン → ステップ5で行う
- GitHub CLI（`gh`）というソフトへのログイン → ステップ6で行う
  （AIが作業した記録をGitHubに残すために必要）
- Claude・OpenAI・Geminiの「APIキー」という鍵 → ステップ8で取得する
  （3つの会社に登録が必要で、使った分だけお金がかかる）

準備するものがai-councilより少し多いが、1つずつこのガイドの通りに
進めれば大丈夫。これから出てくる「コマンド」は、全部この後のステップで
コピーして貼り付けるだけでよい。

---

## ステップ1: プログラムをダウンロードする

1. ブラウザで次のページを開く
   `https://github.com/momokuomomo-crypto/ai-dev-council`
2. 緑色の「Code」ボタンをクリックする
3. 「Download ZIP」をクリックする（これでダウンロードできる。むずかしい
   操作は不要）
4. ダウンロードしたZIPファイルを、いったんデスクトップ（パソコンを
   開いたときに最初に見える画面）に移動する
5. そのZIPファイルを右クリックして「すべて展開」を選ぶ
6. 展開先が「デスクトップ」になっていることを確認して「展開」ボタンを押す
7. デスクトップに`ai-dev-council-main`という名前のフォルダができる。
   これが、これから使うフォルダになる（以下「作業フォルダ」と呼ぶ）

---

## ステップ2: Pythonを準備する

このツールは「Python」というソフトの上で動く。

1. `https://www.python.org/downloads/` を開く
2. 「Download Python」ボタンをクリックしてダウンロードする
3. ダウンロードしたファイルを実行する
   - 画面の一番下にある「Add python.exe to PATH」に必ずチェックを入れる
     （これを忘れると後でエラーになるので注意）
   - あとは「Install Now」を押して進める

---

## ステップ3: 文字を打ち込む画面を開く

いつもはアイコンをクリックしてソフトを使うが、このツールは「文字を
打ち込んで、Enterキーを押す」ことで動かす。その「文字を打ち込む画面」を
開くのがこのステップ。

1. デスクトップにある「作業フォルダ」（ステップ1で作った
   `ai-dev-council-main`フォルダ）を、ダブルクリックで開く
2. フォルダの中身（ファイルやフォルダの一覧）が表示される
3. その一覧の中の、何もない白い部分（ファイルやフォルダの上ではない場所）
   の上で、キーボードの「Shift」キーを押しながら、マウスの右ボタンを
   クリックする
4. 小さいメニューが出てくるので、その中から「PowerShellウィンドウを
   ここで開く」という項目（「ターミナルで開く」という名前のこともある）
   をクリックする
5. 黒い（または濃い青い）画面が新しく開けば成功。これが「文字を打ち込む
   画面」

この画面が開けているかぎり、この画面は「さっきの作業フォルダの中」に
います、という意味になる。これから出てくる命令（英数字の文字列）は、
すべてこの画面にコピーして貼り付け、Enterキーを押して実行していく。

---

## ステップ4: Node.jsを準備する

AIが実際にファイルを作ったりプログラムを実行したりする機能
（「Claude Agent SDK」という部品）は、「Node.js」というソフトの上で
動く。Pythonとは別に、もう1つ準備する必要がある。

1. `https://nodejs.org/` を開く
2. 「LTS」と書かれた方（安定版という意味）のボタンをクリックして
   ダウンロードする
3. ダウンロードしたファイルを実行し、表示される案内の通りに
   「Next」を押して進める（特別な設定は不要）

---

## ステップ5: Claude Code CLIにログインする

AIが実際にプログラムを書く機能は、「Claude Code」というソフトを
経由して動く。すでにClaude Codeを使ったことがある人は、この
ステップは飛ばしてよい。

ステップ3で開いた画面に、次の1行を貼り付けてEnterを押す。

```
npm install -g @anthropic-ai/claude-code
```

インストールが終わったら、次の1行を貼り付けてEnterを押す。

```
claude
```

画面の案内に従ってログインする（ブラウザが開くので、そこで案内通りに
進める）。ログインできたら、いったんこの画面は閉じてよい（ステップ3を
やり直して、新しい画面を開くこと）。

---

## ステップ6: GitHub CLIにログインする

AIが作業した内容の記録は、GitHubというサービスに自動で残される。
そのために「GitHub CLI」（`gh`）というソフトを使う。

1. `https://cli.github.com/` を開く
2. 「Download for Windows」をクリックしてダウンロードし、インストールする
3. ステップ3で開いた画面に、次の1行を貼り付けてEnterを押す

```
gh auth login
```

4. 画面に出てくる質問には、基本的に一番上の選択肢を選んでEnterを押して
   進めればよい（`GitHub.com` → `HTTPS` → `Login with a web browser`
   など）
5. 表示されるコードをコピーし、開いたブラウザの画面に貼り付けてログインする

---

## ステップ7: 必要な部品と、AIの「鍵」を準備する

### 7-1. 必要な部品をインストールする

ステップ3で開いた画面に、次の1行を貼り付けてEnterを押す。

```
pip install anthropic openai google-genai python-dotenv pyyaml claude-agent-sdk==0.2.110
```

### 7-2. APIキー（AIを使うための鍵）を用意する

Claude・OpenAI・Geminiという3つのAIを使うには、それぞれの会社から
「APIキー」という鍵をもらう必要がある。会社ごとにアカウント登録と、
使った分だけお金がかかる支払い設定が必要になる。

- Claude（Anthropic）: `https://console.anthropic.com/`
- OpenAI: `https://platform.openai.com/api-keys`
- Gemini（Google）: `https://aistudio.google.com/apikey`

この部分がいちばんむずかしいので、分からなければ詳しい人に相談すること。

### 7-3. 鍵をファイルに書き込む

1. 作業フォルダの中にある `.env.example` というファイルをコピーする
2. コピーしたファイルの名前を `.env` に変える（`.`から始まる名前ぴったりに）
3. `.env` をメモ帳で開き、`=` の右側にそれぞれの鍵を貼り付けて保存する

```
ANTHROPIC_API_KEY=（ここにClaudeの鍵）
OPENAI_API_KEY=（ここにOpenAIの鍵）
GEMINI_API_KEY=（ここにGeminiの鍵）
```

---

## ステップ8: 実行してみる

### コマンドを実行する

ステップ3で開いた画面に、次のように入力してEnterを押す。
`"　　"`の中身（作ってほしいものの説明）と`--output-dir`のあとの
名前（生成先フォルダの名前）は、自分の好きなものに書き換えてよい。

```
python -m ai_dev_council.pipeline "1から10まで数えるだけの簡単なプログラムを作って" --output-dir ./output/my-first-program
```

### 2回の確認が出る

実行すると、まず「AIに何回アクセスするか」の確認が出る。続けるなら
`y`を入力してEnterを押す（何も入力せずEnterでも「はい」と同じ扱い）。

そのあと、もう1回「これからAIが実際にファイルを作ったりプログラムを
実行したりします」という警告が出る。これも`y`でEnterを押すと、AIが
実際に作業を始める。

しばらく待つ（内容によっては数分かかる）と、`--output-dir`で指定した
フォルダの中に、実際のプログラムファイルが作られる。

---

## ステップ9: 結果を確認する

- `--output-dir`で指定したフォルダの中に、AIが作ったプログラムファイルが
  できている
- 実行の最後に、GitHubのissue（作業記録）へのリンクが表示される。
  ブラウザでそのリンクを開くと、AIがどう設計し、どう確認し合ったかの
  記録が読める

---

## こまったときは

- 「コマンドが見つかりません」と出る
  → ステップ2でPythonの「Add python.exe to PATH」にチェックを
    入れ忘れていないか確認する
- 「'claude' は、内部コマンド...として認識されていません」と出る
  → ステップ5のインストールがうまくいっていない可能性がある。
    ステップ4（Node.js）を先に済ませてからステップ5をやり直す
- 「エラー：本日の実行回数が上限に達しています」と出る
  → 1日に実行できる回数（初期設定では3回）を使い切った。翌日また
    実行できる
- それ以外のエラーが出る
  → 詳しい人に、エラーメッセージをそのまま見せて相談する
